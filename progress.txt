## Codebase Patterns
- ESLint 9 with flat config: use `eslint.config.mjs` (not `.eslintrc.cjs`), `eslint-config-next@16` exports flat config arrays
- Use `tseslint.configs.disableTypeChecked` for `.js`/`.mjs`/`.cjs` files to avoid project service errors
- `eslint-plugin-mdx` requires `import * as mdx` (no default export)
- Prisma typed SQL requires `prisma generate --sql` to regenerate after dependency updates
- Project uses T3 stack: Next.js + tRPC + Prisma + NextAuth
- Lint scripts should exclude non-source dirs (tasks, .agents, prisma, scripts) and *.md files
- Zod v4: use `z.ZodError` (not `ZodError`) and `z.flattenError(err)` (not `err.flatten()`)
- Prisma v7: generator provider is `prisma-client` (not `prisma-client-js`), output goes to `src/generated/prisma/`
- Prisma v7: use `PrismaPg` adapter from `@prisma/adapter-pg` with `{ connectionString }` (not raw string)
- Prisma v7: config uses `migrations.path` (not `migrate.migrations`), no `earlyAccess` property
- Prisma v7: imports from `~/generated/prisma/client` (not `@prisma/client`), SQL from `~/generated/prisma/sql`
- Prisma v7: use extensionless imports (`~/generated/prisma/client` not `client.js`) â€” Turbopack can't resolve `.js` extension to `.ts` files
- react-day-picker v9: classNames use new keys (e.g., `month_caption`, `day_button`, `selected`, `today`, `outside`), `Chevron` component replaces `IconLeft`/`IconRight`
