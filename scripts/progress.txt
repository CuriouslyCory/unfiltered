# URL Table State Progress Log
Started: Wed Mar 11 2026

## Codebase Patterns
- DataTable component is at `src/app/_components/document-table/data-table.tsx` (client component)
- Uses TanStack React Table v8 with `useReactTable` hook
- Next.js App Router: `useSearchParams` requires wrapping in `<Suspense>` in parent server component
- URL state sync pattern: local `useState` for optimistic updates + `useEffect` to sync from `searchParams` on back/forward
- Column definitions are in `columns.tsx`, sortable columns: dateSigned, riskScore, title, type, updatedAt
- Project uses 4-space indentation (prettier `--tab-width 4`)
- Quality checks: `pnpm lint` (next lint + tsc --noEmit)
- Debounced URL sync: use `useRef<ReturnType<typeof setTimeout>>` for timeout ID, clear on each change, push URL after delay
- Column filters are managed externally via `columnFilters` state (not via table's internal setFilterValue) for URL sync consistency
- Pagination URL param is 1-indexed (user-facing), internal TanStack state is 0-indexed; page 1 omits the param
