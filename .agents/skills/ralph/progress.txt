## Codebase Patterns
- Header component is at src/app/_components/theme/header.tsx (client component with mobile menu state)
- Layout metadata is in src/app/layout.tsx using Next.js Metadata export
- Uses shadcn/ui components from ~/components/ui/ and app-level components from src/app/_components/
- Tailwind theme tokens: text-muted-foreground, border-border, bg-muted, bg-accent for consistent theming
- The project uses a `~/` path alias that maps to `src/`
- Dev server: `pnpm dev` on port 3000
- Typecheck: `npx tsc --noEmit`
- Document types are defined in Prisma schema as enum: EXECUTIVE_ORDER, FACT_SHEET, REMARKS, LEGISLATION, OTHER
- Human-readable type labels are in src/app/_components/document-type-badge.tsx (typeConfig map)
- shadcn UI components installed via: `pnpm dlx shadcn@latest add <component>`
- Admin document detail page: src/app/admin/documents/[id]/page.tsx (server component)
- Document editor: src/app/admin/_components/document-editor.tsx (client component)
- Document utilities: src/lib/document-utils.ts (artifactStyles, artifactOrder, getArtifactStyle, getArtifactByTitle, RISK_RANGES)
- Artifact section IDs follow pattern: `artifact-${title.toLowerCase().replace(/\s+/g, "-")}` — used for scroll targeting
- Scroll-to-section pattern: src/app/eo-summary/[slug]/_components/sticky-document-header.tsx
- IntersectionObserver pattern: src/app/eo-summary/[slug]/_components/sticky-document-header.tsx (lines 29-47)
- Risk color coding: src/app/_components/risk-score.tsx
- Environment variables: src/env.js using @t3-oss/env-nextjs createEnv
- tRPC mutations in document-editor.tsx all call router.refresh() on success
- Badge component: ~/components/ui/badge with variants: default, secondary, destructive, outline
- Card + Collapsible components available in ~/components/ui/
- Tooltip provider wrapped at layout level
