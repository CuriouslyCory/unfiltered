## Codebase Patterns
- Project uses ESM (`"type": "module"`) with `tsx` for dev, TypeScript target ES2024
- Path alias `~/*` maps to `./src/*` in tsconfig
- Formatting uses 4-space indentation (prettier `--tab-width 4`)
- Pre-existing `model-tracker.test.ts` is an example file, not a real test - excluded in vitest config
- Zod updated to 3.25.x to satisfy both AI SDK (`^3.25.76`) and LangChain (`^3.24.x`) peer deps
- esbuild added to `pnpm.onlyBuiltDependencies` for vitest support
- `@ai-sdk/google` v3.x: use `createGoogleGenerativeAI({ apiKey })` factory, not `google(model, { apiKey })` - the `google()` shorthand only takes model ID
- `@ai-sdk/groq`: use `createGroq({ apiKey })` factory for API key configuration
- vitest path aliases: add `resolve.alias` in vitest.config.ts to match tsconfig `~/*` path alias
- AI SDK v6 tool() uses `inputSchema` (not `parameters`) and `execute` - the `parameters` field is from older v3 API
- AI SDK Tool type has `execute` as optional - use `!` non-null assertion in tests when calling `tool.execute!()`
- Convert LangChain array exports (e.g., `dateTimeTools = [...]`) to record objects (e.g., `dateTimeTools = { name: tool, ... }`)
- Tools imported in `src/tools/index.ts` need `as unknown as StructuredToolInterface` casts until US-011 migrates index.ts
- For mocking PrismaClient in vitest, use `vi.hoisted()` to define mock instances so they're available in hoisted `vi.mock()` factories
- PrismaClient must be mocked as a class (not plain object) since `database.ts` uses `new PrismaClient()`
- LangChain `config?.configurable` pattern (RunnableConfig) has no AI SDK equivalent - convert configurable fields to explicit tool input parameters
- DynamicStructuredTool converts identically to regular tool(): `func` → `execute`, `schema` → `inputSchema`
- LangChain community tool class instances (e.g., DuckDuckGoSearch, WikipediaQueryRun) need full reimplementation with direct API calls, not just format conversion
- `duck-duck-scrape` package: use `search(query, { safeSearch })` - returns `{ results: [{ title, url, description }] }`
- `ollama-ai-provider` 1.x exports `EmbeddingModelV1` but AI SDK v6 `embed()`/`embedMany()` expects `EmbeddingModel` (V2/V3) - use `as unknown as EmbeddingModel` cast
- Mock QdrantClient as a class (not vi.fn()) in vitest since it's instantiated with `new`
- When removing the old `getVectorStore` from qdrant.ts, re-export the new one from `~/utils/vector-store` since `memories.ts` imports it from `./qdrant`
- VectorStore.delete() accepts both `string[]` and `{ ids: string[] }` for backward compat with LangChain callers
