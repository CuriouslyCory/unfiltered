## Codebase Patterns
- Output validation: validateWorkflowOutput(workflow, content) in src/utils/output-validation.ts returns { valid, errors[] }
- SSE streaming: use reply.hijack() to prevent Fastify from auto-closing, then write to reply.raw with writeHead() and write()
- Clean up event listeners and intervals on request.raw 'close' event for SSE connections
- Workflow names are defined as a const array with z.enum for Zod schema validation
- `WorkflowSchema` uses `z.enum()` for runtime validation of workflow names
- The `Workflow` type is inferred from `z.infer<typeof WorkflowSchema>`
- `app.workflowEvents` (EventEmitter) is decorated onto FastifyInstance in document.ts
- electron-log is used for logging throughout (import log from "electron-log")
- All workflow handlers follow pattern: `handle<Name>Workflow(app: FastifyInstance, documentId: number)`
- WorkflowEvents type in document.ts defines typed event signatures; import Workflow type from run-all.ts for event data
- WORKFLOW_ARTIFACT_TITLE maps workflow names to DB artifact titles (enhance excluded — updates multiple artifacts)
- Prisma compound unique: `documentId_title` on DocumentArtifact for fetching specific artifacts
- Artifact titles: Research, Key Points, Constitutional Considerations, ELI5, Risk Score Details, Honest Title, Letter of Concern, Social Post, Take Action
